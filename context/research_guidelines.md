# Research Phase Guidelines

These guidelines define the success criteria and expected depth for each research phase in the AutoScout Discovery Pipeline.

## 1. Idea Phase
- **Focus**: Divergent thinking and creative hypothesis generation.
- **Success Criteria**: 
  - At least 3 distinct hypotheses.
  - Each hypothesis must have a clear "Testability" section.
  - Hypotheses must avoid repeating common knowledge (verified via web search).

## 2. Method Phase
- **Focus**: Convergent design and procedural planning.
- **Success Criteria**:
  - A step-by-step protocol for testing the selected hypothesis.
  - Explicit list of required tools, datasets, or technical stacks.
  - Identification of variables or metrics to be measured.

## 3. Experiment Phase
- **Focus**: Execution analysis and implementation notes.
- **Success Criteria**:
  - Detailed technical outline (not executable code, but deep architecture).
  - Feasibility assessment (Red/Amber/Green status).
  - List of "Lessons Learned" and technical risks.

## General Principles
- **Conciseness**: Avoid fluff; focus on high-signal technical content.
- **Adversarial Flow**: The Reviewer and Hater roles MUST challenge the status quo. 
- **Sequentiality**: Each phase must explicitly build on the markdown output of the previous phase.
