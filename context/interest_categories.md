# Interest Category Definition Schema

To maintain high signal content, all categories must follow this structure:

## 1. Category Blueprint
- **Title**: A concise name (e.g., "AI Infrastructure").
- **Keywords**: Semantic anchors for search queries.
- **Focus Areas**: Specific sub-topics of interest.
- **Exclusions**: Topics to ignore specifically within this category.

## 2. Dynamic Categories
Categories are managed dynamically. When the **Interest Profiler** detects a drift in user interest (via uploaded documents or search history), it should propose updates to this file.

## 3. Seed Categories
- **Finance**: Macro trends, quantitative trading, and fintech innovation.
- **AI Research**: LLM architecture, agentic workflows, and efficient inference.
- **Personal Productivity**: Tooling, specialized knowledge management, and cognitive science.
